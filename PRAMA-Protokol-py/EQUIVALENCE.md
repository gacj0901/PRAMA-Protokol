## Identity anchors

The current state machine is pinned by two independent JSON vector sets:

- `../tests/data/v0_3_golden_vectors_v1.json` exercises activation, debt,
  capacity bounds, controls, warm-up, missing-value failure and causal `G`;
- `../tests/data/v0_3_golden_vectors_v2.json` exercises logical-order periodic
  ring rebuilds, including warm-up and multiple window boundaries.

The v2 vectors are generated by a standalone `decimal.Decimal` oracle that
imports neither production implementation. The compatibility projection is
separately pinned by `tests/golden_gamma.npz`.

## Required identity

Certification requires:

- exact row counts and temporal indices;
- bit identity between batch and streaming inside each language;
- exact Boolean and integer fields across languages;
- continuous outputs within the declared ULP contract;
- an unchanged compatibility golden-vector trajectory;
- fail-closed errors that leave streaming state unmodified.

## Numerical falsifiers

The long-run audit independently reconstructs the trailing margin sum and the
capacity ledger. Two mutations must be detected:

1. omission of the accumulated-debt coupling;
2. disabling periodic logical-order ring rebuilds.

A certification that cannot distinguish the protected coupling from
floating-point tolerance is informational, never passing.

## Autonomy

No downstream package is a reference implementation. Comparing the kernel
with an implementation that imports it would be circular. The golden vectors,
Python reference and Rust core contained in this repository are the complete
executable identity chain.

## Change control

Any recurrence, alignment, warm-up, dtype or output-semantics change requires
a version change, append-only anomaly entry, deliberate vector update and
joint Python/Rust recertification.

Equivalence certifies implementation identity, not empirical usefulness.
